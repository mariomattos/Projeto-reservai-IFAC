Requisitos Não Funcionais

01 - Usabilidade (O sistema é totalmente intuitivo e de fácil compreensão, portanto o usuário não precisa ter muito conhecimento à respeito de tecnologias para usufruir das funcionalidades que o aplicativo oferece).

02 - Tecnologia Para Dispositivos Móveis (A aplicação foi desenvolvida em React Native. A pertinência dos dados foi realizada no sistema de gerenciamento de banco de dados não relacional Firebase, sem a necessidade de compra de licenças com alguns desses serviços).

03 - Armazenamento (O banco de dados, implementado em Firebase, armazenará toda as informações necessárias que forem inseridas na aplicação).

04 - Disponibilidade (O sistema estará disponível 24 {vinte e quatro} horas por dia para download no Google Play Store). 

05 - Portabilidade (A aplicação é suportada em dispositivos móveis com sistema operacional Android 4.2. A portabilidade pode ser expandida).

06 - Segurança (Os dados gerados pela aplicação, a base de dados e o servidor são seguros, via autenticação no sistema).

07 - Distribuição (O sistema estará disponível para download na loja Google Play a custo zero).

08 - Eficiência (O sistema utiliza os recursos da melhor forma possível. Não ocupa muito espaço nos smartphones dos usuários, e dentro dos requisitos de software e hardware não apresenta problemas relativos ao seu uso).

09 - Eficácia (O sistema cadastra os dados do usuário. Apresenta a listagem de usuários e suas características. Além disso, possibilita o usuário reservar espaços do IFAC. Assim, alcança os objetivos propostos).

10 - Efetividade (Não existe no IFAC nenhum aplicativo de reserva de espaços que possua as mesmas funcionalidades. Dessa forma, e de acordo com os testes realizados, os usuários se sentirão mais confortáveis para reservar locais com facilidade e poder utilizá-los como bem entender).
